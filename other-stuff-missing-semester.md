## Git and Github

[git](https://git-scm.com/) is a command-line tool
used to store and track changes to source code
(programming text) repositories and related
documentation. It comes pre-installed on MacOS,
available through the Terminal application, and
within VSCode, both in the integrated terminal and
in the "Source Control" view.

[Github](https://github.com/) is a popular website based
on the git tool that allows easily publishing and sharing
git repositories, and also provides web-based collaboration
features. Most major open-source sofware systems are hosted
on Github, including
[Lean itself](https://github.com/leanprover/lean4)
and its body of theorems,
[Mathlib](https://github.com/leanprover-community/mathlib4).

Both of these would be useful to a working mathematician,
or any other persion doing technical work with tools that
read or manipulate text files.

Knowing just a small bit about these tools, even just a few
basic operations, will go a long way in allowing you to be
able to manage and permanently store and share your work.

[Intro to github and git](https://docs.github.com/en/get-started/start-your-journey/about-github-and-git

## Markdown (Github flavor)

Markdown is an open standard for authoring and viewing rich text
documents where the content is also easily readable and writable
in plain text form. Markdown is not quite fully standardized in
all aspects, so it has various variants or "flavors". Here we'll
concentrate on Github's flavor since that's what people will see
when viewing your Markdown files on Github.

Read the [intro to Github's Markdown flavor](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax)

Visual Studio code also recognizes Markdown files for file
names ending with `.md`. When editing Markdown, a toolbar
button to the right of the open tabs (with a magnifying
glass over a divided page) should allow showing a preview
of the rendered/formatted Markdown. Or `Shift-CMD-V` (Mac)
will show a preview in a new tab. A few Github-specific
features won't render properly within VSCode, but this is
usually not a problem since Markdown is always human-readable
even without rendering to rich text.
